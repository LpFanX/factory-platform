# Changelog

## v0.1 — 2026-07-09

Первый рабочий срез платформы: полный цикл «идея → конвейер агентов → ревью человеком»
на живом стенде (factory.lpfanx.ru), с контролем стоимости и автообновлением движка.

### Платформа
- FastAPI-бэкенд: REST + WebSocket-стрим стадий прогона, SQLite-персистентность
  (прогоны/активность), настройки в JSON, отдача SPA.
- React SPA (VK Sans, светлая tech-тема): живой конвейер агентов (React Flow + Framer
  Motion), вкладки-роуты `/` `/runs` `/budget` `/approvals` `/settings` `/logs`.
- Банк идей: добавление/одобрение/отклонение из UI, единая очередь с движком.
- Согласования: предпросмотр черновика (markdown), «принять / доработать», копирование.
- Страница «Расходы»: живой баланс AITunnel, расход день/месяц, стоимость прогонов,
  разбивка по моделям, алерт низкого баланса.
- Авторизация (опциональная): HMAC-cookie сессии, login/logout в SPA, защита WebSocket;
  тумблер `deploy/factory-auth.sh on|off`.

### Интеграция с движком content-agents (≥ 4.57)
- Движок — read-only git-клон с автообновлением по таймеру; данные строго в
  `CONTENT_AGENTS_DATA` (клон всегда git-чистый).
- Пер-ролевые модели через env: Perplexity `sonar` на research/fact-check (нативный
  поиск; обёртка снимает function tools — sonar их не поддерживает), Claude Sonnet на
  тексты, Opus на финал, gpt-4o-mini на SEO-углы.
- Бюджеты контекста стадий (`PROMPT_CONTENTS_STAGE_INPUT_BUDGET`).

### Контроль стоимости
- Жёсткий кап стоимости прогона: preflight-оценка до вызова + пост-фактум учёт по
  `usage.cost_rub` шлюза; стоимость печатается даже при падении прогона.
- `max_tokens` на вызов, лимит tool-итераций, промпт-кэш AITunnel (`session_id` +
  `cache_control`). Ориентир себестоимости: 150–250 ₽/статья.

### Автономность
- systemd-таймеры: автообновление движка + автономный тик (одобренная идея → прогон →
  SQLite → идея done); автопилот включается в настройках, по умолчанию выключен.

### Деплой
- systemd-юниты + nginx (TLS, WebSocket, SPA fallback), `.env` через EnvironmentFile.
- README с заметками для переноса в Kubernetes (PVC, git-sync движка, egress, CronJob).

### Известные ограничения
- Публикация в CMS (Strapi) — не подключена (read-адаптер уже есть в движке; write-путь
  и сетевой доступ из контура — следующий этап).
- VK Workspace бот — задел в roadmap, не реализован.
- Полный прогон целиком (все стадии до финала) на боевом шлюзе ещё не подтверждён замером —
  оборван капом на writer; фактическая цена статьи уточнится после контрольного прогона.
